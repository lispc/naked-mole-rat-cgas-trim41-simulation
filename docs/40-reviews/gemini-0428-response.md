### 1.1 突变序列执行错误 — ❌ 误报 (FALSE ALARM)

**Gemini 指控**：`Hsap_cGAS_4mut.fasta` 中 D431S 突变实际发生在 D436，真正的 D431 仍是 Aspartate。

**核查方法**：
```python
# 正确解析 FASTA（跳过 header 行）
wt  = ''.join([l.strip() for l in open('sequences/Hsap_cGAS_WT.fasta') if not l.startswith('>')])
mut = ''.join([l.strip() for l in open('sequences/Hsap_cGAS_4mut.fasta') if not l.startswith('>')])

diffs = [(i+1, wt[i], mut[i]) for i in range(len(wt)) if wt[i] != mut[i]]
# 结果: [(431, 'D', 'S'), (479, 'K', 'E'), (495, 'L', 'Y'), (498, 'K', 'T')]
```

**结论**：四个突变位点 **100% 正确**。Gemini 极可能犯了与 Kimi 首次验证时相同的错误——将 FASTA header 行误纳入序列，导致编号偏移和伪差异。Hgal 4mut_rev 同理验证通过（S463D, E511K, Y527L, T530K）。

**影响**：基于这些序列的 MD 和 US **无需因序列错误而重做**。

### 1.2 跨体系对比逻辑失效 — ⚠️ 部分成立

**Gemini 指控**：`scripts/compare_systems.py` 直接按残基编号相减，Hgal/Hsap 存在 32 aa 系统性偏移，导致 ΔRMSF 错位。

**代码审查**：

| 函数 | 使用场景 | 处理方式 | 结论 |
|------|----------|----------|------|
| `compare_systems.py::compare_rmsf()` | 任意两系统对比 | 长度不同时用 `interp1d` 插值后相减 | ⚠️ **有风险**：若用于跨物种，插值基于 resid 值而非序列比对，物理位置会错位 |
| `compare_four_systems.py::plot_intrinsic_rmsf_delta()` | 仅限 intra-species（WT vs mutant） | 同一物种 resid 完全一致，直接相减 | ✅ **正确** |
| `compare_four_systems.py::compare_active_sites()` | 四系统 active site 对比 | 显式同源映射：`Hgal S463↔Hsap D431`, `E511↔K479` 等 | ✅ **正确** |

**结论**：核心比较逻辑（`compare_four_systems.py`）已正确处理同源映射。但 `compare_systems.py` 作为通用两两对比工具，**若被误用于跨物种比较**，确实会产生无物理意义的 ΔRMSF。

**行动**：在 `compare_systems.py` 中增加 guard——检测到 resid 范围/长度显著不匹配（如相差 >30）时抛出警告或拒绝执行。

### 1.3 活性位点编号"幻觉" — ⚠️ 部分成立

**但**：实际生成的 `Hsap_WT_summary.json` 中 active_sites 使用的是 **D431, K479, L495, K498**（Hsap 序列编号），对应距离合理（29–44Å）。说明**实际分析数据是正确的**，问题出在 README.md 的示例命令上。

**行动**：
1. 修正 README.md 示例，明确区分 "序列编号" 与 "PDB/Amber 拓扑编号"
2. Docking 完成后重新构建 Hgal 系统时，用 `build_system_v2.py` 输出实际 Amber resid，消除 +19 偏移
