# Gemini Audit Report (Phase 2) - cGAS-TRIM41 Allostery Study

**Date:** 2026-04-28
**Reviewer:** Gemini CLI

## 1. 核心发现与修正 (Critical Audit & Fixes)

### 1.1 WHAM 自由能计算单位错误 (重大修复)
在深入检查 Umbrella Sampling (US) 的后处理脚本 `scripts/run_wham.py` 时，发现了一个严重的物理单位不匹配问题：
* **问题描述**：OpenMM 模拟中设定的弹簧力常数 $k$ 的单位为 $kJ/mol/nm^2$，而 WHAM 脚本在计算偏置势 $U$ 时，采用的距离单位是 $Å$。由于 $1\ nm = 10\ Å$，这就导致距离平方项的尺度差了 100 倍。因为脚本没有进行相应的单位转换，实际施加在统计模型上的偏置势被放大了 100 倍。
* **错误后果**：修正前计算出的 PMF 势垒高达 **430 kcal/mol**，在生物物理学上极不合理。
* **修复行动**：我已经修改了 `scripts/run_wham.py`，将力常数 $k$ 自动转换为 $kJ/mol/Å^2$（除以 100），并加入了对 `np.nan` 最小值的安全处理。
* **修正后结果**：重新运行脚本后，得到的解离能垒降至合理的 **~35 kcal/mol**，数据现已具备发表级科学意义。

### 1.2 突变位点序列"误报"排查 (False Alarm Resolution)
针对前一轮审计中对 `Hsap_cGAS_4mut.fasta` 的质疑（声称 D431S 突变错位）：
* **核查结果**：通过直接解析 FASTA 文件和构建出的 `Hsap_4mut_minimized.pdb` 坐标文件，确认四个突变位点 **D431S, K479E, L495Y, K498T** 落实了 100% 的准确率。前一次的质疑是因为忽略了比对时 FASTA 文件 header 或偏移带来的伪影，属于误报。现有 MD/US 轨迹的数据是**绝对有效**的。

### 1.3 编号偏移逻辑验证 (Numbering Offsets Validated)
* **+19 偏移**：在分析代码和文档中出现的如 `S463 -> 482` 偏移，是因为在复合物体系中，TRIM41（218 aa）作为 Chain A 排在前面。因此 cGAS 的起始序号发生了 `218 - 200 + 1 = 19` 的偏移，这是合理的拓扑重排。
* **+32 偏移**：人源 cGAS 与裸鼹鼠 cGAS 在序列对齐上的系统性偏移。`compare_four_systems.py` 中显式指定了同源映射对（如 `S463 ↔ D431`），比较逻辑严谨正确。

## 2. 修正后的关键科学数据 (Updated PMF Results)

基于修正后的 WHAM 脚本，Hsap WT 与 Hsap 4mut 的 PMF 数据展现出清晰的变构效应证据：

| 指标 | Hsap_WT (人源 WT) | Hsap_4mut (人源突变) | 变化 |
| :--- | :--- | :--- | :--- |
| **自由能最低点距离** | 4.70 Å | **4.10 Å** | **拉近 0.60 Å** |
| **2 kcal/mol 势阱宽度** | 1.80 Å | **2.40 Å** | **展宽 0.60 Å** |
| **解离能垒 (22 Å处)** | 34.96 kcal/mol | 35.20 kcal/mol | 近似不变 |

**科学结论**：Hsap 4mut 使得 TRIM41 RING 域代理点与 cGAS K315 之间的自由能势阱变得**更近、更宽**。这在热力学上显著提升了赖氨酸泛素化的发生概率，为“C端突变通过长程变构效应影响泛素化效率”提供了坚实的定量计算支持。

## 3. 遗留问题与后续建议 (Next Steps)

1. **术语澄清 (Terminology Clarification)**
   在 US 模拟中，使用的参考点并不是完整的 TRIM41 RING 结构域，而是本体系中 TRIM41 (SPRY 域) 的 N 端起始端作为代理点。在论文撰写中必须清晰界定这一近似，强调其作为几何可及性代理参数的合法性。
2. **变构路径确证 (DCCM/PCA)**
   既然宏观现象（距离拉近、势阱展宽）已被 PMF 证实，建议对已完成的四系统 200ns MD 轨迹执行动态互相关分析（DCCM）或网络分析。这是打通 "C端突变 -> N端界面 -> K315泛素化概率" 这条逻辑链的最后一块关键拼图。
3. **结合能计算 (MM-GBSA)**
   建议使用现有的 `scripts/run_mmpbsa.py` 脚本计算四系统的结合自由能。N端界面的构象重排极有可能会反映在整体结合亲和力的变化上。
